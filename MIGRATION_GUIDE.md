# Captain System Migration Guide

> **From:** Nomaan's laptop (WSL 2 + Docker Desktop, Windows 11)
> **To:** Standalone tower (fresh Windows 11, dedicated 24/7 trading machine)
> **Date created:** 2026-04-07
> **Total time estimate:** 1-2 hours (most of it waiting for Docker builds)

---

## How to Use This Guide

This guide is designed to be followed **top to bottom, step by step**. Don't skip ahead.

Each section is marked with one of two labels:

- **MANUAL** — You must do this yourself (Windows GUI, typing passwords, etc.)
- **CLAUDE CODE** — Paste the provided prompt into Claude Code and let it run. The prompt will handle everything until it hits something that needs your input.

**Important:** Do the LAPTOP steps first (Stage 1), then move to the TOWER (Stages 2-7).

---

## What Gets Migrated

| What | How | Size |
|------|-----|------|
| All source code, configs, P1/P2 data, scripts | `git clone` (everything tracked in git) | ~90 MB |
| QuestDB database (29 tables of bootstrapped state) | USB drive / network copy | ~205 MB compressed |
| Redis snapshot | USB drive / network copy | ~34 KB |
| QDB export CSVs | USB drive / network copy | ~344 KB |
| `.env` secrets file | Re-create manually on tower | N/A |
| Windows Task Scheduler tasks | Re-create manually on tower | N/A |
| `.wslconfig` | Re-create manually on tower | N/A |

**Not migrated (rebuilt automatically):**
- Docker images (rebuilt on `--build`)
- `journal.sqlite` files (recreated on startup)
- `node_modules/` (rebuilt inside Docker)
- `__pycache__/` (regenerated by Python)
- `backups/` directory (fresh backups created on tower)

# STAGE 2 — TOWER: Windows Setup

Everything in this stage is done on the **new tower**, in **Windows** (not WSL yet).

## Step 2.1 — Disable Sleep and Hibernate

> **MANUAL**

1. **Open Settings** > System > Power & battery (or Power & sleep)
2. Set **Screen timeout** to: `Never` (or your preference)
3. Set **Sleep** to: `Never`
4. **Open Command Prompt as Administrator** (right-click Start > Terminal (Admin))
5. Run these commands one at a time:

```cmd
powercfg /hibernate off
powercfg /setactive SCHEME_MIN
powercfg /change standby-timeout-ac 0
powercfg /change monitor-timeout-ac 0
```

**What these do:**
- `hibernate off` — Prevents the PC from writing RAM to disk and shutting down
- `SCHEME_MIN` — Sets power plan to "High Performance"
- `standby-timeout-ac 0` — Disables automatic standby (0 = never)
- `monitor-timeout-ac 0` — Disables monitor sleep (0 = never)

**Verify:** Run `powercfg /getactivescheme` — should show "High Performance" or "Ultimate Performance"

**If you don't see "High Performance":**
```cmd
powercfg /duplicatescheme e9a42b02-d5df-448d-aa00-03f14749eb61
powercfg /setactive e9a42b02-d5df-448d-aa00-03f14749eb61
```

## Step 2.2 — Disable Automatic Windows Updates During Market Hours

> **MANUAL**

1. Open **Settings** > Windows Update > Advanced options
2. Set **Active hours** to: 9:00 AM - 6:00 PM (covers NY session)
3. Turn on **Notify me when a restart is required** (so it never auto-reboots)

**Why:** Windows Update can reboot your PC mid-session. Active hours prevent this.

## Step 2.5 — Create .wslconfig

> **MANUAL**

1. Open **Notepad** (or any text editor)
2. Paste the following (adjust `memory` and `processors` based on your tower's specs):

```ini
# .wslconfig for Captain System tower
# Adjust memory to ~80% of your total RAM

[wsl2]
memory=32GB
processors=8
swap=8GB
localhostForwarding=true
networkingMode=mirrored
```

**How to decide values:**
- **memory:** If your tower has 32 GB RAM, set to `32GB`. If 64 GB, set to `48GB`. If 16 GB, set to `12GB`.
- **processors:** Set to your total CPU core count (check Task Manager > Performance > CPU > "Cores")

3. Save as: `C:\Users\<YOUR_USERNAME>\.wslconfig`
   - In Notepad: File > Save As > navigate to `C:\Users\<YOUR_USERNAME>\` > filename: `.wslconfig` > Save as type: "All Files"
   - **Important:** Make sure it's `.wslconfig` not `.wslconfig.txt`

4. **Restart WSL** to apply:
```cmd
wsl --shutdown
```
Then reopen Ubuntu from Start menu.

**Verify:** Inside WSL, run `free -h` — should show your configured memory limit.

| Error | Fix |
|-------|-----|
| `Cannot connect to the Docker daemon` | Open Docker Desktop on Windows first, wait 30 seconds |
| `vm.max_map_count: Permission denied` | Run: `sudo sysctl -w vm.max_map_count=1048576` then re-run captain-start.sh |

| `port 80 already in use` | Something else is using port 80. Run: `sudo lsof -i :80` to find it |
| `Error response from daemon: network not found` | Run: `docker network prune -f` then re-run |
| Build hangs on `pip install` | Docker might be running out of memory. Increase `.wslconfig` memory. |

## Step 6.1 — Create CaptainSleepGuard Task

> **MANUAL**

   - Name: `CaptainSleepGuard`

   - Click **New...**
   - Begin the task: On a schedule
   - Daily, recur every 1 day
   - Start: `09:00:00 AM` (adjust to your timezone — this should be ~30 min before NY market open in your local time)
   - Enabled: checked
5. **Actions tab:**
   - Click **New...**
   - Action: Start a program
   - Program: `powershell.exe`

   - (Replace `<YOUR_USERNAME>` with your Windows username)
6. **Conditions tab:**
   - Uncheck "Start the task only if the computer is on AC power"
7. **Settings tab:**
   - Check "Allow task to be run on demand"
   - Check "If the task fails, restart every: 5 minutes" (up to 3 times)
8. Click **OK**

# Troubleshooting Reference

## Docker won't start in WSL

```bash
# Check if Docker Desktop is running on Windows side
docker info

# If not, start Docker Desktop from Start menu, wait 30 seconds, try again
```

## QuestDB data looks empty after restore

```bash
# The QuestDB data directory needs correct ownership
# Stop containers, check permissions, restart
cd ~/captain-system
docker compose -f docker-compose.yml -f docker-compose.local.yml down
ls -la questdb/db/  # should show directories

# Restart — QuestDB will read the files
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d
```

If still empty, re-run bootstrap:
```bash
docker compose -f docker-compose.yml -f docker-compose.local.yml exec -T \
  -e PYTHONPATH=/app \
  -e BOOTSTRAP_ACCOUNT_ID=20319811 \
  -e BOOTSTRAP_USER_ID=primary_user \
  -e BOOTSTRAP_STARTING_CAPITAL=150000 \
  captain-offline \
  python /captain/scripts/bootstrap_production.py
```

## TopstepX authentication fails

1. Verify API key is correct in `.env`
2. Make sure the key was generated from the tower's browser (not the laptop)
3. Check logs: `docker compose -f docker-compose.yml -f docker-compose.local.yml logs captain-command --tail 50`
4. Try regenerating the API key on TopstepX

## Port 80 already in use

```bash
# Find what's using it
sudo lsof -i :80

# Common culprits: Apache, IIS, Skype
# On Windows: check IIS is not installed (it uses port 80)
# Settings > Apps > Optional Features > remove "Internet Information Services"
```

## Containers keep restarting

```bash
# Check logs for the crashing container
docker compose -f docker-compose.yml -f docker-compose.local.yml logs captain-online --tail 100

# Common causes:
# - Missing .env variables
# - QuestDB not ready (increase start_period in docker-compose.yml)
# - Memory limit too low (increase in docker-compose.local.yml)
```

## WSL runs out of memory

Edit `.wslconfig` on Windows side:
```
C:\Users\<USERNAME>\.wslconfig
```
Increase the `memory` value, then:
```cmd
wsl --shutdown
```
Reopen Ubuntu and restart Captain.

## GUI shows blank page

```bash
# Rebuild the GUI container
docker compose -f docker-compose.yml -f docker-compose.local.yml up -d --build captain-gui
# Then restart nginx to pick up new files
docker compose -f docker-compose.yml -f docker-compose.local.yml restart nginx
```
